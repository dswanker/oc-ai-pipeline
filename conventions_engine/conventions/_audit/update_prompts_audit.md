**`DVS_TRANSLATE_PROMPT` — active, no engine relevance.** Used at pipeline.py:2138 inside Path C (edited DVS → translate to XLSForm changes). The prompt takes "Current XLSForm JSON" + "DVS Changes" and returns updated XLSForm JSON. The XLSForm is a downstream build artifact, not a Study Spec — the conventions engine has already finished its work before Path C runs. Correctly outside the engine's scope.

**`SPEC_FROM_BUILD_PROMPT` — dead, planning artifact.** Defined to reverse-engineer a Study Specification from built XLSForm files. If wired, it would presumably trigger from Path B (Edited Build ZIP uploaded). Path B's docstring explicitly says "skip steps 1-6, run 7 only" — the actual implementation chose NOT to reconstruct Study Spec from a Build ZIP. The prompt was reserved during planning but the feature was never built. No engine relevance because the feature itself doesn't exist.

**`GENERATE_STUDY_SPEC_PROMPT` — dead, refactor relic.** Defined to drive Claude-side generation of Study Spec PDF + XLSX from JSON. Phase C.1 moved this work to local scripts (`skills/protocol-analysis/scripts/generate_study_spec_pdf.py` + `generate_study_spec_xlsx.py`), invoked synchronously from `run_study_spec_files()` without going through Claude. The prompt is left over from the pre-local-script architecture.

**`GENERATE_PROTOCOL_SUMMARY_PROMPT` — dead, refactor relic.** Same pre-Phase-C.1 era as #3, for Protocol Summary PDF generation. Now handled by `skills/protocol-analysis/scripts/generate_protocol_summary_pdf.py` (or equivalent local script). Note: Protocol Summary is a separate document from Study Spec; even if this prompt were active, it would not be engine-relevant because the engine operates on Study Spec JSON only.

**`QUOTE_PDF_FROM_XLSX_PROMPT` — dead, deprecated path.** Defined to regenerate Quote PDFs from edited Quote XLSX uploads (Path D). The pipeline docstring explicitly marks Path D as "DEPRECATED, logs a message + skips" — the entire feature was retired. Pricing/quote flow is outside the Study Spec engine boundary regardless.
